#  Техническое задание: Admin Web Panel
# Веб-панель администратора для управления контентом

## 1. Назначение

Админ-панель используется для:
-  Управления книгами (CRUD)
-  Редактирования текста и метаданных
-  Загрузки и управления аудиофайлами
-  Настройки тайм-маркеров для синхронизации
-  Управления пользователями
-  Просмотра статистики и аналитики
-  Модерации контента

---

## 2. Архитектура

**Тип:** SPA (Single Page Application)
**Framework:** React 18+
**Паттерн:** MVVM / Component-based
**Язык:** JavaScript

### Технологический стек

#### Core
- **React** 18+
- **JavaScript**
- **Vite**

#### UI Framework
- **Material-UI (MUI)** или **Ant Design**

#### Forms
- **React Hook Form** - управление формами

#### Rich Text Editor
- **Draft.js** или **Slate**
- **TinyMCE** - альтернатива

#### Audio
- **Wavesurfer.js** - визуализация аудио
- **Howler.js** - воспроизведение

#### HTTP Client
- **Axios** с interceptors
- **React Query** для кеширования

#### Routing
- **React Router v6**

#### Testing
- **Jest** - unit тесты
- **React Testing Library**
- **Cypress** - E2E тесты

## 4. Основные экраны

### 4.1 Dashboard (Главная)
**URL:** `/`

Отображает:
-  Статистика (кол-во книг, пользователей, глав)
-  Графики активности
-  Популярные книги
-  Требующие внимания (книги без аудио, ошибки синхронизации)

### 4.2 Books Management
**URL:** `/books`

#### 4.2.1 Books List
- Таблица всех книг
- Фильтры (язык, автор, статус)
- Поиск по названию/автору
- Сортировка
- Пагинация
- Действия: Просмотр, Редактирование, Удалить

#### 4.2.2 Book Editor
**URL:** `/books/:id/edit` или `/books/new`

Форма с полями:
- Название книги *
- Автор *
- Описание
- Язык *
- Обложка (загрузка изображения)
- Статус (Активна / Черновик)

Кнопки:
- Сохранить
- Отменить
- Удалить (если редактирование)

### 4.3 Chapters Management
**URL:** `/books/:bookId/chapters`

#### 4.3.1 Chapters List
- Список глав книги (с drag-and-drop для изменения порядка)
- Добавить главу
- Для каждой главы:
  - Порядковый номер
  - Название
  - Длина текста
  - Статус аудио (Есть / Нет / Обрабатывается)
  - Действия: Редактировать, Удалить

#### 4.3.2 Chapter Editor
**URL:** `/chapters/:id/edit`

**Вкладки:**

**1. Текст**
- Rich text editor для текста главы
- Форматирование (жирный, курсив, абзацы)
- Автосохранение каждые 30 сек
- История изменений (версионирование)

**2. Аудио**
- Загрузка аудиофайла (drag-and-drop)
- Предпросмотр: Wavesurfer визуализация
- Управление воспроизведением
- Информация об аудио (длительность, формат, размер)

**3. Синхронизация (Time Markers)**
- Два панели:
  - Левая: Текст главы
  - Правая: Аудио плеер с визуализацией
- Создание тайм-маркеров:
  - Выделить текст
  - Указать позицию в аудио
  - Создать маркер
- Таблица маркеров с действиями (редактировать, удалить)
- Тестирование синхронизации (проиграть и показать подсветку)

### 4.4 Users Management
**URL:** `/users`

- Таблица пользователей
- Фильтры (роль, дата регистрации)
- Поиск по email/username
- Действия:
  - Просмотр профиля
  - Изменить роль (USER ↔ ADMIN)
  - Заблокировать/Разблокировать
  - Удалить

### 4.5 Settings
**URL:** `/settings`

Настройки:
- Общие настройки приложения
- Интеграция с TTS сервисами
- Хранилище файлов (S3)
- Email настройки
- Параметры безопасности

## 5. Ключевая функциональность

### 5.1 Загрузка файлов

#### Обложки книг

#### Аудиофайлы
- Поддержка форматов: MP3, M4A, WAV
- Максимальный размер: 500 MB
- Progress bar при загрузке
- Проверка валидности (длительность, битрейт)
- Автоматическая конвертация (опционально)

### 5.2 Rich Text Editor

Функции:
- Форматирование текста (bold, italic, underline)
- Абзацы и переносы строк
- Undo/Redo
- Подсчет символов/слов
- Автосохранение

### 5.3 Time Markers Editor

Workflow:
1. Админ загружает аудио
2. Открывает редактор синхронизации
3. Слушает аудио
4. Сохраняет все маркеры

### 5.4 Audio Waveform Visualization

Функции:
- Визуализация аудио волны
- Маркеры на волне
- Клик по волне → добавить маркер
- Zoom in/out
- Play/Pause
- Seek

### 5.5 Preview Mode

Тестирование синхронизации:
- Режим предпросмотра чтения
- Воспроизведение аудио с подсветкой текста
- Проверка всех маркеров
- Отладка несоответствий

## 6. Нефункциональные требования

### 6.1 Производительность
-  Быстрая загрузка страниц (<2 сек)
-  Lazy loading для списков
-  Code splitting
-  Оптимизация bundle size

### 6.2 UX
-  Responsive design (работа на планшетах)
-  Интуитивный интерфейс
-  Подсказки и tooltips
-  Уведомления об успехе/ошибках (toast notifications)

### 6.3 Надежность
-  Автосохранение данных
-  Подтверждение удаления (модальные окна)
-  Rollback изменений
-  Валидация форм
-  Обработка ошибок сети

### 6.4 Безопасность
-  Доступ только для ADMIN роли
-  JWT аутентификация
-  Защита от XSS
-  CSP (Content Security Policy)
